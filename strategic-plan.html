Executive Summary

The University of Rochester Medical Center Data & Analytics Master Plan establishes a comprehensive framework for building an enterprise analytics ecosystem that enables data-driven decision making across clinical, operational, and research domains. This three-year strategic roadmap focuses on leveraging best-in-class technologies to create a federated analytics development model while maintaining enterprise governance and standards.

Key Technologies

  • Informatica: Enterprise data integration, quality, and governance platform
  • Denodo: Data virtualization layer for federated queries and access
  • Tableau: Enterprise visualization and analytics development platform

Strategic Benefits

  • Federated Model: Balances centralized governance with domain-specific expertise
  • Integration: Unified view of data across clinical, financial, and operational systems
  • Self-Service: Enables analytics capabilities across skill levels and departments

Implementation Focus

  • Data Foundation: Establishing reliable, governed data sources
  • Developer Community: Building federated development capabilities
  • Workflow Integration: Embedding analytics in operational processes

Implementation Roadmap

Our three-year implementation roadmap outlines a phased approach to building our enterprise analytics capabilities, starting with fundamental infrastructure and governance, then expanding to enable federated development, and finally scaling to enterprise-wide adoption and innovation.

Phase 1: Foundation (Q2-Q3 2025)

Current Phase

Key Initiatives

  • Epic Caboodle optimization and performance tuning
  • Denodo platform implementation and data source connections
  • Initial Tableau development standards and templates
  • ServiceNow analytics request process implementation
  • Data governance framework establishment

Key Outcomes

  • Improved Caboodle refresh times and model optimization
  • Core virtualized data layer for clinical analytics
  • Standardized development environment for Tableau
  • Centralized request intake and tracking process
  • Initial data governance structure and roles
Phase Progress 65%

Phase 2: Enablement (Q4 2025 - Q1 2026)

Future Phase

Key Initiatives

  • Metric Insights implementation and portal design
  • Federated developer community establishment
  • Department-level Tableau developer training program
  • Slicer Dicer adoption and template library
  • Initial clinical intranet analytics integration

Key Outcomes

  • Enterprise metrics portal with initial dashboards
  • First cohort of trained federated developers
  • Analytics development community of practice
  • Clinical self-service analytics adoption increase
  • Workflow-embedded analytics prototypes
Phase Progress 10%

Phase 3: Scale (Q2-Q4 2026)

Future Phase

Key Initiatives

  • Enterprise-wide Metric Insights rollout
  • Advanced data integration patterns with Informatica
  • Expanded federated development community
  • Clinical intranet analytics expansion
  • Advanced Epic Caboodle data modeling

Key Outcomes

  • High engagement with centralized analytics portal
  • Complex data integration capabilities across systems
  • Decentralized analytics development at scale
  • Analytics embedded in critical clinical workflows
  • Enhanced data modeling for specialized use cases
Phase Progress 0%

Phase 4: Innovation (2027+)

Future Phase

Key Initiatives

  • Advanced analytics and predictive modeling
  • AI/ML healthcare use cases development
  • Closed-loop analytics with automated workflows
  • Enhanced personalization and adaptive insights
  • Enterprise-wide data literacy program

Key Outcomes

  • Predictive analytics embedded in clinical workflows
  • AI-driven clinical decision support models
  • Automated interventions based on analytics
  • Personalized analytics delivery by role/need
  • Data-driven culture across the organization
Phase Progress 0%
